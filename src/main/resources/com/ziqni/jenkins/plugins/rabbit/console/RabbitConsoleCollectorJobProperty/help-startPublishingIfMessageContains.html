Start Publishing if Message Contains

This function inspects text being printed to the console and triggers publishing if a specified case-sensitive string is found.

Purpose: The function enables conditional publishing of console output to an AMQP exchange. When the specified string is detected in the console output, it initiates publishing of all subsequent messages, including the message that triggered the action.

Configuration:

  • Case-Sensitive String: Provide the exact string that will trigger publishing. The function will look for this string in a case-sensitive manner.

Example Usage: If the string "ERROR:" is specified, the function will start publishing all console messages once a line containing "ERROR:" appears.

This feature is useful for starting conditional publishing based on specific log outputs, such as errors or significant events, without flooding the AMQP channel with unnecessary logs.
